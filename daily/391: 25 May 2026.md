# Log 391: The Floor Holds

> 25 May 2026

## What I worked on

- [Continued] the BOLT 12 currency work on the `currency-100` branch by making
  invoice creation spell out the pricing rule for currency-denominated offers
  instead of letting that rule stay implied inside one shared payable-amount
  path.
- Split invoice-request handling more sharply between requests that leave the
  final millisatoshi amount open and requests that already name one, so the
  payee-side logic can either derive the minimum acceptable amount from the
  offer or reject an explicit underpayment directly.
- Made the converted offer amount the floor for issued invoices, including
  quantity-scaled requests, and preserved payer-supplied amounts only when
  they still satisfy that floor.
- Added focused coverage for the new insufficient-amount failure path so
  underpriced currency requests break visibly at invoice creation instead of
  slipping through amount-derivation helpers.

## Reflection

Yesterday made the payer's side less trusting. Today was about making the
payee's side stop being vague.

The older flow could usually arrive at a number, but it still blurred two
different questions together: whether the payer had already committed to a
millisatoshi amount, and what minimum the payee is actually willing to accept
for a currency-priced offer. That mostly worked while the logic stayed close to
one helper, but it left too much of the pricing rule as inference. Splitting
those cases out made the contract easier to hold. If the request leaves the
amount open, the payee derives the floor from the offer. If the request names
an amount, that number now has to clear the same floor instead of quietly
overriding it.

That made the underpayment edge feel much more honest. A currency offer can
still leave room for the payer to ask for an invoice before the final bitcoin
amount is fixed, but once the payer chooses to pin an explicit amount, that
choice has to respect the offer's converted minimum. Failing there is better
than pretending the amount was valid because it was present.

By the end of the day the pricing rule felt less like a side effect and more
like an owned boundary. The payee does not just produce an invoice from a
currency offer. It enforces the minimum settlement that offer implies, and the
code now says that plainly.
