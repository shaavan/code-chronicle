# Log 394: Re-Landing

> 29 May 2026

## What I worked on

- [Continued] the BOLT 12 currency work on the `currency-104` branch by
  rebasing the series onto the newer offers tree and rethreading the currency
  path through the current interfaces instead of forcing the older patch shape
  back into place.
- Updated the surrounding `ChannelManager` scaffolding and carried the same
  fixes through helpers, fuzz targets, and tests so the rebased branch sat
  cleanly on the newer base after the `BestBlock` to `BlockLocator` changes.
- Tightened the currency types by making `ExchangeRate` compare by value,
  hiding `ExchangeRange` field access behind a checked constructor and
  accessors, and stopping invalid ranges from being assembled casually at call
  sites.
- Adapted the series to the newer metadata-carrying offer flows so invoice and
  refund builders, invoice-request contexts, static invoices, and blinded
  payment paths all keep `payment_metadata` inside the main offers path instead
  of drifting into special handling.
- Split converted-amount validation more precisely between values that cannot
  be verified, values that fall below the acceptable minimum, and values that
  exceed the acceptable maximum.

## Reflection

This day was less about pushing the feature forward and more about seeing
whether it could survive the codebase it now had to live in.

The earlier currency work had been built against an older offers tree. Rebase
pressure exposed how much of that shape belonged to the moment it was written
in. Some of the work was mechanical, but not in the trivial sense. Each place
the branch no longer fit forced a choice between preserving old structure out
of habit and letting the newer offers model decide what belongs. Reworking the
surrounding `ChannelManager` paths, the helpers, and the tests made the branch
feel less like a preserved patch set and more like code trying to earn its
place on the current tree.

That same pressure made the currency types hard to ignore. `ExchangeRange`
still trusted callers too much, and equal exchange rates still had too much
room to behave like distinct values depending on how they were spelled. Closing
those gaps mattered for the same reason the rebase mattered. The feature could
not keep leaning on local understanding at every boundary and still claim to be
settled. The newer offer flows pushed in the same direction. Once
`payment_metadata` had to move through invoice creation, refunds, static
invoices, and blinded payment paths, it became much harder to let currency
support sit off to the side as its own exception.

By the end of the day, the work felt less like reviving an older branch and
more like forcing it through current architectural pressure until only the
parts that still made sense were left standing.
