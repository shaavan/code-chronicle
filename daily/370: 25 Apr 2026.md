# Log 370: Reading It from Outside

> 25 April 2026

## What I worked on

- Reviewed the Q8 OpenSats report end to end and rewrote it as a document for
  an evaluator rather than for my own memory of the quarter.
- Removed repeated claims across sections and quarters, tightened the
  introduction, and rewrote the recurrence section around the now-settled split
  between `rust-lightning` and LDK Node.
- Reframed the dummy-hop section around what actually changed this quarter
  instead of re-claiming baseline work that had already landed earlier.
- Pulled Offers currency support out into its own smaller thread, cleaned up
  the developer-facing API section, and updated the quarter comparison and
  next-quarter goals to match the stronger structure.
- Aligned the use-of-funds wording with the prior quarter and rewrote the
  conclusion around the quarter's actual result and the convergence it now
  points toward.

## Reflection

Today was report work, but it felt closer to design review than to writing.

The first draft still carried too much of my own proximity to the quarter. It
kept trying to restate background I already knew, which made some sections pull
against each other. The recurrence section was one example. Until I rewrote it
around the `rust-lightning` versus LDK Node split as it now actually stands, it
was still carrying more motion than conclusion. The dummy-hop section had a
similar problem. It was too willing to count previously merged groundwork as if
it belonged to this quarter's result.

Once I started reading the document from outside that working context, the edit
pressure got simpler. Each section needed one job. It had to say what moved,
what that movement now means, and where the ownership or implementation
constraint really sits. That made it easier to cut repeated claims, separate
Offers currency support into a smaller secondary thread, and stop the API and
comparison sections from trying to do explanatory cleanup they no longer
needed.

The useful part of the day was not that the report became shorter. It was that
it became harder to hide inside. By the end, the document was doing less
reconstruction and more accounting. It now reads much more like a quarter that
produced a few real positions and a narrower set of next steps, which is what I
needed it to say in the first place.
