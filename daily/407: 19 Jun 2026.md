# Log 407: Failure by Class

> 19 June 2026

## What I worked on

- Continued the dummy-hop follow-up on the `dummy-follow-26` branch by working
  through the remaining `cargo test -p lightning` fallout as grouped failure
  classes instead of chasing them file by file.
- Rewired forwarded-event expectations so
  `expected_forwarded_extra_fees` once again participates in
  `PaymentForwarded.total_fee_earned_msat` checks, and corrected one async
  payment test that had also been missing the first blinded-tail forwarding
  fee.
- Fixed held-HTLC timeout coverage that was still mining only to the old
  pre-dummy-hop CLTV threshold, which had been making the LSP fail earlier than
  the updated path now should.
- Cleaned up an expired static-invoice path that was still asserting on an old
  log message even though the real invalid-onion failure was already the right
  one.
- Finished with the remaining breakage reduced to one-hop blinded payment and
  offer or refund tests that still assume pre-dummy-hop exact fee and amount
  behavior, leaving one last coherent expectation class to repair.

## Reflection

Once the shared helper pass landed, today stopped feeling like feature work and
started feeling like triage with better structure. That helped more than I
expected. Reading failures by class made it easier to see which ones were
actually connected and which ones only happened to fall over in the same test
run.

The forwarded-fee fixes were the first good example. Those failures were not
saying anything new about the runtime path. The harness had simply drifted out
of contact with what the updated event surface was now reporting. The timeout
and static-invoice failures felt different for exactly that reason. They were
about path timing and observability after dummy hops changed the tail, not
about accounting, and they became easier to fix once I stopped treating them
as part of the same problem.

What I liked by the end was how much narrower the remaining uncertainty had
become. The last failures now sit in one place: tests that still want exact
pre-dummy-hop amounts and exact sent-fee behavior, even though bucketed relay
parameters and rounding have made those paths slightly looser. That is a much
better place to stop than a wall of unrelated red.
