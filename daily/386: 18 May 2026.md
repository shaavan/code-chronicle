# Log 386: Only What Was Named

> 18 May 2026

## What I worked on

- [Continued] the BOLT 12 currency work on the `currency-98` branch by
  separating the amount explicitly encoded in an `InvoiceRequest` from the
  amount that still has to be derived later from the offer and quantity.
- Reworked the request-side helpers so downstream code stops inferring intent
  indirectly through `has_amount_msats`-style checks and instead goes through
  one payable-amount path for missing, unsupported, and overflow cases.
- Kept the newer `CurrencyConversion` entry point but removed the extra
  `MsatsRange` layer once it became clear it was only restating a small piece
  of conversion logic the amount model could carry directly.
- Moved accepted exchange-rate bounds closer to `Amount` so offer and invoice
  code now resolve currency amounts into millisatoshi ranges where those values
  are actually consumed instead of teaching the rest of the stack another
  intermediate helper type.

## Reflection

Today felt mostly like taking staging back out of the amount model.

The earlier currency work had already started making room for application-owned
conversion, but the request API was still blurring two different kinds of
state. One amount was something the payer had actually named in the request.
The other only existed after combining the offer, quantity, and conversion
rules. Keeping those under the same accessor made too much of the later logic
guess at intent instead of reading it directly from the type.

Splitting that line made the rest of the cleanup much easier to trust. Once
request data stayed distinct from payable amount, invoice creation and
validation could settle missing, unsupported, and overflow cases through one
path instead of each carrying a local fallback. Dropping `MsatsRange` came from
the same instinct. The extra abstraction sounded tidy, but it was mostly
another place for the model to explain itself before doing real work. By the
end of the day, the improvement was not that currency support looked broader.
It was that the amount model had stopped saying more than it actually knew.
