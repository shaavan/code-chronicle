# Log 385: Something That Holds

> 16 May 2026

## What I worked on

- [Continued] the BOLT 12 recurrence work on the `rust-recurrence` branch by
  finishing the earlier refactor into one stack that now treats offer,
  `InvoiceRequest`, and invoice recurrence as the same model instead of
  separate local approximations.
- Moved basetime resolution into request verification and the respond path so
  recurrence gets settled where the request contract is checked instead of
  being reconstructed later from invoice-local state.
- Replaced the older loose invoice recurrence fields with one structured object
  and cleaned up the surrounding parsing, serialization, stale builder paths,
  and macro callsites until the feature compiled honestly again.
- Filled in the missing public API docs around the new recurrence surface so
  the refactor stopped failing on `deny(missing_docs)` once the types became
  part of the exported shape.
- Broadened the tests beyond the immediate feature path by adding invoice
  recurrence parsing and rejection coverage, then used wider runs to catch and
  close one remaining invoice-creation dependency on unfinished
  recurrence-token plumbing before opening the first recurrence [PR].

## Reflection

Today felt like the point where the recurrence work had to stop being a
promising refactor and become something that could actually stand on its own.

The earlier passes had already pushed the request side into a more honest
shape, which made the remaining invoice-local leftovers harder to ignore. Too
much of the behavior was still split across parsing, construction, and ad hoc
field handling, so the model kept leaking different versions of itself
depending on which path touched it. Finishing this pass was mostly about
removing that split. Once offer recurrence, request recurrence, and invoice
recurrence all started describing the same thing, the rest of the cleanup got
much less mysterious.

The tests helped in a useful way here because they stopped me from calling the
job done at the compiler boundary. Narrow feature fixes would have left the
last invoice-creation gap buried until later. Wider coverage flushed it out
immediately, and closing it made the whole stack feel more real than the green
unit tests alone would have. By the end of the day, opening the PR did not
feel like marking a checkpoint. It felt like the first part of the recurrence
series had finally become coherent enough to be reviewed as one thing.
