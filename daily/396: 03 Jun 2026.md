# Log 396: Quoted Edges

> 03 June 2026

## What I worked on

- [Continued] the BOLT 12 currency work on the `currency-107` branch by
  replacing the older "expanded exchange-rate range" model with a tighter
  representation built from one quoted rate plus explicit lower and upper
  tolerances.
- Moved the logic that turns tolerance policy into accepted exchange-rate
  bounds into one place, so callers no longer have to pass around derived
  minimum and maximum rates without owning how those numbers were formed.
- Refined converted-amount validation near `MAX_VALUE_MSAT` so an oversized
  upper bound is clamped instead of rejected outright, while the lower bound
  still has to resolve to a real payable amount.
- Propagated the new model through offer and invoice builders,
  `ChannelManager`, and the surrounding test and fuzz helpers, then fixed the
  derived-key invoice-response coverage in `c_bindings` so that path stayed in
  step with the builder changes.

## Reflection

This day felt like choosing what the conversion model should admit plainly
instead of letting it stay half-buried in derived numbers.

The earlier shape worked, but it asked too much trust from the callers.
Providers handed back a fully expanded range, and by the time that range moved
through the rest of the code, the tolerance policy that produced it had mostly
disappeared from view. Reworking that around one quoted rate with explicit
lower and upper tolerances made the interesting part visible again. It also
made the model feel more honest. If the real rule is "accept this quote within
these bounds," then that should be the thing the code carries around.

The same kind of tightening showed up at the payment limit boundary. Treating
an oversized upper end as outright invalid was technically safe, but it was
too blunt for what the code was trying to prove. What mattered was whether the
accepted range still contained some real payable amount. Once that became the
question, clamping the upper side to `MAX_VALUE_MSAT` felt more accurate than
rejecting the whole conversion for a theoretical value we would never pay.

Most of the rest of the day was the usual test of whether a cleaner idea stays
clean once it leaves the place it was invented. Threading the new shape
through builders, `ChannelManager`, fuzz helpers, and `c_bindings` was less
dramatic work, but it mattered because it showed the model was not only nicer
to describe. It could hold its shape across the stack.
