# Weekly Log 09: Locking It In

> 27 Feb 2026 – 02 March 2026

## Overview

This week brought the currency conversion work to a stable and merge-ready
state. The focus shifted from feature implementation to API clarity, logical
alignment, and safety guarantees. Several refinements tightened the structure,
simplified usage, and closed important correctness gaps.

## Highlights

### API Consolidation and Testing

- Completed end-to-end currency conversion testing across the codebase.
- Introduced a new `InvoiceRequest::payable_amount_msats` API to centralize
  currency conversion and amount validation logic before returning a value.
- Simplified `InvoiceRequest::amount_msats()` so it now strictly returns the
  explicitly set amount on the invoice request, removing implicit inference from
  the underlying Offer.
- Reduced API ambiguity and clarified responsibility boundaries, making correct
  usage the natural path rather than an opt-in pattern.  
  ([Currency updates][Currency58])

### Logical Tightening Before PR

- Revisited validation placement to ensure correctness checks cannot be
  accidentally bypassed during invoice creation.
- Cleaned up inter-code coupling and clarified the role of each introduced API
  before preparing the branch for PR.
- Closed a crucial safety gap by ensuring that a node can only create
  currency-denominated Offers in currencies it explicitly supports, preventing
  potential misuse or self-inflicted inconsistencies.

## Reflections

This week was less about adding new capabilities and more about making the
system hold under scrutiny.

Testing surfaced opportunities to simplify. Simplification exposed hidden
ambiguities. Addressing those ambiguities made the model clearer and safer. By
the end of the week, the currency support reads as a coherent structure rather
than a layered addition.

The final validation check — restricting Offer creation to supported currencies
— was small in code, but significant in intent. It reflects a broader principle:
correctness should be enforced by design, not assumed by discipline.

The branch now feels ready.
