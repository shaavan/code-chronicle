# Log 335: Closing the Footgun

> 02 Mar 2026

## What I Worked On

- [Added] a missing validation check for currency-denominated Offers

## Reflections

While reviewing the latest currency conversion changes, I noticed an important
gap: we were not ensuring that a node can only create an Offer in a currency it
actually supports.

Without this check, it would be possible for a node to create Offers in
arbitrary currencies — even ones it cannot interpret or convert correctly.
That’s a quiet but serious footgun, and exactly the kind of thing that’s better
prevented by structure than left to user discipline.

I added a validation step to ensure that currency-denominated Offers can only be
created when the node explicitly supports the corresponding currency. This makes
the behavior safer by default and aligns Offer creation with the node’s actual
capabilities.

A small change, but an important one for correctness.
