# Log 363: Where It Belongs

> 13 April 2026

## What I worked on

- [Continued] the currency conversion work and resolved key correctness issues.
- Fixed validation, range-handling, and fuzz-alignment problems across the branch.
- Turned back to recurrence and narrowed Phase 1 to the generic seams needed for
  a node-owned lifecycle.
- Defined the boundary for that move without growing the existing recurrence PoC
  further inside `ChannelManager`.

## Reflection

Today had two distinct kinds of pressure, and both mattered for the same
reason: they were about putting responsibilities back where they belong.

On the currency side, I addressed the remaining open comments and fixed a set
of important issues. Range handling now behaves safely under tolerance,
validation is enforced consistently at the right boundary, and the fuzz path is
aligned with the actual amount-resolution logic.

That work was satisfying partly because it removed ambiguity. The branch now
feels less like something that merely passes review and more like something
whose edge cases tell the same story as its happy path.

From there, I turned back to recurrence.

What became clearer today was that the real problem is not how to extend the
current PoC, but how to stop extending it in the wrong place. If recurrence is
going to live comfortably in `LDK Node`, then `rust-lightning` needs to expose
the right protocol seams and then get out of the way. That reframed the work
from "add recurrence support" to "define the minimum surface that makes moving
it out actually possible."

I spent time sitting with that transition and broke it down into a Phase 1
plan. The focus is not to implement recurrence yet, but to make it possible:
manual inbound request handling, public response/payment primitives, and a
clean signer boundary. All without growing the existing PoC logic further.

By the end, the path forward felt narrower in a good way. The uncertainty was
no longer about direction. It was about execution.

Tomorrow is for turning that seam plan into real code.
